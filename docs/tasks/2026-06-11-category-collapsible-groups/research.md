# Research: 기준정보관리 > 카테고리 — 대분류별 접기/펼치기

- 날짜: 2026-06-11
- 요청 원문: 기준정보관리> 카테고리에서 대분류 , 소분류를 대분류 별로 접었다 펼칠수 있도록 해줘

## 요약

기준정보 관리 화면의 카테고리 탭은 `frontend/src/pages/SettingsPage.tsx`의 `CategoriesTab` 컴포넌트(46-216행)가 담당한다. 현재는 평탄한 테이블 한 장에 모든 카테고리 행을 나열하고, 같은 대분류가 연속되면 첫 행에만 대분류명을 표시하고 나머지는 `·`로 대체하는 의사(pseudo) 그룹 표시만 있다(122-145행). 카테고리 데이터는 백엔드가 `kind → major → minor` 순으로 정렬해 내려주므로(`backend/app/routers/categories.py:14-18`) 같은 (구분, 대분류) 행은 항상 연속으로 도착한다 — 프런트에서 추가 정렬 없이 연속 구간을 그룹으로 묶을 수 있다.

해결 방향: `CategoriesTab` 내부에서 categories 배열을 (kind, major) 단위로 그룹핑하고, 각 그룹마다 "대분류 헤더 행"(클릭 시 접기/펼치기 토글, 셰브론 아이콘 + 대분류명 + 구분 배지 + 소분류 개수)을 렌더링한 뒤 펼침 상태일 때만 소분류 행들을 표시한다. 접힘 상태는 컴포넌트 로컬 상태(`(kind, major)` 키 기반)로 관리하면 충분하며, CRUD 후 `fetchAll()`로 배열이 통째로 교체되어도(`frontend/src/stores/masterData.ts:84-95`) 이름 기반 키이므로 상태가 유지된다. 백엔드·스토어·타입 변경은 불필요한 순수 프런트 UI 변경이다.

## 관련 파일 및 근거

- `frontend/src/pages/SettingsPage.tsx:46-216` — 수정 대상인 `CategoriesTab` 컴포넌트. 테이블 렌더링(110-148행)과 `·` 의사 그룹 로직(125-132행)이 여기 있다.
- `frontend/src/pages/SettingsPage.tsx:2` — lucide-react 아이콘 import 지점. 셰브론 아이콘(ChevronDown/ChevronRight 등)을 여기서 추가하면 된다.
- `frontend/src/stores/masterData.ts:49-56, 84-95` — categories 배열 출처. 모든 CRUD 후 `fetchAll()`로 전체 재조회되어 배열이 교체된다 → 접힘 상태를 배열 인덱스가 아닌 (kind, major) 이름 키로 관리해야 하는 근거.
- `backend/app/routers/categories.py:12-18` — `/categories`가 `kind, major, minor` 순 정렬로 응답. 같은 (kind, major) 행이 항상 연속임을 보장하는 근거.
- `frontend/src/types.ts:28-36` — `Category` 타입(`major`, `minor`, `kind`, `nature`). 변경 불필요.
- `frontend/src/components/ui/table.tsx` — 사용 중인 테이블 프리미티브. 별도 collapsible UI 컴포넌트는 `frontend/src/components/ui/`에 존재하지 않으므로, 헤더 행 클릭 + 조건부 렌더링으로 구현한다(라이브러리 추가 불필요).

## 영향도

- 없음 (SettingsPage.tsx의 `CategoriesTab` 내부만 변경) — `CategoriesTab`은 `SettingsPage`에서만 사용되며(547행), 스토어·API·타입·다른 페이지는 건드리지 않는다. 카테고리 데이터를 쓰는 다른 화면(TransactionsPage, BudgetsPage 등)은 스토어를 직접 구독하므로 영향 없음.

## 성공 기준 (Acceptance Criteria)

- [ ] AC-1: 카테고리 탭에서 각 (구분, 대분류) 조합마다 그룹 헤더 행이 1개씩 표시되고, 헤더에 대분류명·구분(수입/지출) 배지·소분류 개수가 보인다 — 브라우저에서 기준정보 관리 > 카테고리 탭을 열어 육안 확인.
- [ ] AC-2: 그룹 헤더 행을 클릭하면 해당 대분류의 소분류 행들이 숨겨지고(접힘), 다시 클릭하면 표시된다(펼침). 셰브론 아이콘 방향이 상태에 따라 바뀐다 — 브라우저에서 클릭 토글로 확인.
- [ ] AC-3: 한 그룹을 접어도 다른 그룹의 펼침 상태는 변하지 않는다(그룹별 독립 상태) — 그룹 2개 이상에서 하나만 접고 나머지가 유지되는지 확인.
- [ ] AC-4: 최초 진입 시 모든 그룹은 펼쳐진 상태다(기존처럼 전체 목록이 보임) — 페이지 새로고침 후 확인.
- [ ] AC-5: 접힘 상태에서 카테고리 추가/수정/삭제를 수행해도(스토어 fetchAll로 목록 재조회) 변경되지 않은 그룹의 접힘/펼침 상태가 유지된다 — 한 그룹을 접은 뒤 다른 그룹에 소분류를 추가하고 접힘 상태가 풀리지 않는지 확인.
- [ ] AC-6: 소분류 행의 수정(연필)·삭제(휴지통) 버튼과 추가 다이얼로그 등 기존 CRUD 기능이 변경 전과 동일하게 동작한다 — 추가/수정/삭제 각 1회씩 수행해 확인.
- [ ] AC-7: 이름이 같은 대분류라도 구분(수입/지출)이 다르면 별개 그룹으로 취급된다 — 수입/지출 양쪽에 같은 대분류명을 만들어 독립 토글되는지 확인.
- [ ] AC-8: `frontend`에서 타입 체크/빌드(`npm run build`)가 오류 없이 통과한다.

- [ ] 접힘 상태를 `(kind, major)` 문자열 키 기반 컴포넌트 로컬 상태로 관리 (기본값: 펼침). 키 자료구조(Set/Record)는 구현 재량.
- [ ] 기존 `·` 의사 그룹 표시 로직(125-132행)을 제거하고, 그룹 헤더 행(셰브론 + 대분류명 + 구분 배지 + 소분류 개수, 클릭 토글) + 펼침 시 소분류 행 렌더링으로 교체.
- [ ] lucide-react에서 셰브론 아이콘 import 추가. 헤더 행 전체 클릭 영역, hover 커서 등 상호작용 스타일은 구현 재량.
- [ ] 소분류 행에서 대분류 열을 어떻게 표시할지(빈 칸/들여쓰기)는 구현 재량 — 단 소분류명·구분·성격·행 액션은 기존과 동일하게 유지.

## 미해결 질문

- "전체 접기/전체 펼치기" 일괄 버튼은 요청에 없어 범위에서 제외했다. 필요하면 후속 요청으로 처리.
- 접힘 상태의 영속화(localStorage 등)는 요구되지 않아 컴포넌트 로컬 상태로 한정했다. 탭 전환(Tabs는 언마운트 방식) 시 상태가 초기화될 수 있는데, 이는 허용 범위로 본다 — 유지가 필요하면 사용자 결정 필요.
